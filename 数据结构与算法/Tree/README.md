树结构一定要是一颗排序树,否则查询就没意义了,最常用的排序规则就是左小右大.
只有维持了排序的树的查询效率才是log(n)的,不管几叉树.
树结构最重要的就是让变动操作(增加/删除)尽量能保持一个低的高度log(n),然后查询才能log(n).
排序树的增加操作只需要在叶子结点后添加新节点即可,
先做一个查询找出那个合适的叶子结点(叶子结点从左至右是排序的),然后在该叶子节点后面增加新节点即可.
树最麻烦的操作是删除,它需要分多种情况去处理,主要是子树的处理.

平衡树:
其实树结构最重要的一件事就是: 维护树的 排序 + 低高度(平衡树)
增加和删除操作需要保持树的高度始终是log(n) -> 平衡树
红黑树和B树等都属于平衡树,可以保持树的结构平衡.

树结构玩的就是平衡树: 如何在增加和删除操作后维护树的平衡 ?!
